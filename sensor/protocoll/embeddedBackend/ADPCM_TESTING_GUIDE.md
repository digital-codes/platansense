## Sensor Protocol Usage

### Basic ADPCM Upload

The sensor always sends ADPCM format:

```python
from protoEngine import ProtoEngine

# Initialize sensor
pt = ProtoEngine("network.ssid", "http://localhost:8000", 1, "sensor_key")
pt.setDebug(True)

# Connect and authenticate
pt.connect()
pt.join()

# Load ADPCM file (sensor format)
with open("test.adpcm", "rb") as f:
    audio_data = f.read()

# Upload ADPCM data
resp = pt.upload(audio_data, format="adpcm")

# Check response
if resp.get("status") == "ok":
    transcription = resp.get("transcription")
    classification = resp.get("classification")
    response = resp.get("response")
    conv_id = resp.get("conversation_id")
    conv_reset = resp.get("conversation_reset")
    
    print(f"Transcription: {transcription}")
    print(f"Classification: {classification}")
    print(f"Response: {response}")
    print(f"Conversation ID: {conv_id}")
    print(f"Reset: {conv_reset}")
```

### Command Line Usage

## ADPCM Format Details

### Sensor Specifications

- **Format**: IMA ADPCM
- **Sample Rate**: 8000 Hz
- **Channels**: Mono
- **Bits per Sample**: 16-bit (before compression)
- **Compression**: ~4:1 ratio

### Audio Quality

The sensor works best with:
- Clear German speech
- No background noise
- Moderate speaking speed
- Complete sentences (e.g., "Hallo, wie geht es dir?")
- Short utterances (2-5 seconds)

### File Sizes

Typical ADPCM file sizes:
- Short phrase (1-2s): ~2-4 KB
- Medium sentence (3-4s): ~4-8 KB
- Long statement (5-8s): ~8-16 KB
- Maximum allowed: 512 KB

## Conversation Tracking

The backend tracks conversation state and provides this information:

### Response Format

```json
{
  "uuid": "Sensor_1_abc123def456",
  "status": "ok",
  "transcription": "Hallo, wie geht es dir?",
  "classification": ["personal", "nature"],
  "response": "Mir geht es gut! Ich bin eine Platane in Karlsruhe.",
  "audio_played": true,
  "conversation_id": "conv_a1b2c3d4e5f6_1234567890",
  "conversation_reset": true,
  "conversation_timestamp": 1234567890,
  "message_count": 1
}
```

### Conversation Lifecycle

```
Upload 1 → conv_id: abc123, reset: true, count: 1  (New conversation)
↓
Upload 2 → conv_id: abc123, reset: false, count: 2 (Continue)
↓
Upload 3 → conv_id: abc123, reset: false, count: 3 (Continue)
↓
"Stop" / Timeout → conv_id: def456, reset: true, count: 1 (New conversation)
```

### Reset Conditions

Conversations reset automatically when:
1. User says "stop" (case-insensitive)
2. No interaction for 2 minutes (configurable)

### Tracking Conversation Changes

```python
# Track conversation changes across uploads
previous_conv_id = None

for i in range(5):
    with open(f"audio_{i}.adpcm", "rb") as f:
        audio_data = f.read()
    
    resp = pt.upload(audio_data, format="adpcm")
    
    current_conv_id = resp.get("conversation_id")
    conv_reset = resp.get("conversation_reset")
    
    # Detect new conversation
    if current_conv_id != previous_conv_id:
        print(f"🆕 New conversation: {current_conv_id}")
        previous_conv_id = current_conv_id
    
    if conv_reset:
        print(f"🔄 Conversation was reset")
```

## Troubleshooting

### ADPCM File Issues

**Problem**: File won't transcribe
```
Solution:
1. Check audio quality (should be clear German speech)
2. Verify file size (2-16KB optimal)
3. Ensure format is true IMA ADPCM
4. Test with known good file
```

**Problem**: No transcription returned
```
Solution:
1. Check Whisper installation and model
2. Verify audio format (8000Hz mono)
3. Test audio playback quality
4. Check backend logs for errors
```

### Conversation Tracking Issues

**Problem**: Conversation ID is None
```
Solution:
1. Check backend is sensorRagUpload.php (not old sensorUpload.php)
2. Verify response format includes conversation fields
3. Check backend error logs
```

**Problem**: Unexpected conversation resets
```
Solution:
1. Check timeout configuration (default: 2 minutes)
2. Verify "stop" command isn't being misdetected
3. Check network stability
```

### Environment Issues

**Problem**: Piper TTS not found

Solution:

## Performance Characteristics

### Processing Time (per ADPCM upload)

- Network upload (2-8KB): < 0.1s
- ADPCM decoding: < 0.1s  
- Whisper transcription: 2-5s
- LLM classification + response: 1-3s
- TTS synthesis + playback: 2-4s
- **Total**: 5-12s per interaction

### Memory Usage

- Sensor client: ~10-20MB
- Backend processing: ~100-200MB (LLM dependent)
- Temporary files: Cleaned automatically

### Network Requirements

- Upload bandwidth: Minimal (< 10KB per request)
- Latency tolerance: Can handle 50-100ms latency
- Connection: Stable connection preferred for conversation continuity

## Security Considerations

### Audio Privacy

- All audio processing happens server-side
- Temporary audio files cleaned after processing
- Conversation state stored securely in JSON files

### Conversation Security

- Conversation IDs are randomly generated
- Time-based IDs prevent prediction
- Timeout prevents session hijacking
- Reset only via "stop" command or timeout

### Transport Security

- JWT authentication for all requests
- Challenge-response during authentication
- Encrypted device keys
- HTTPS recommended for production

## Best Practices

### Audio Quality

1. **Clear Speech**: Ensure German is clearly spoken
2. **Noise Reduction**: Minimize background noise
3. **Appropriate Length**: 2-5 seconds optimal
4. **Complete Sentences**: Use full sentences for best transcription

### Conversation Management

1. **Track resets**: Monitor conversation_reset flag
2. **Handle timeouts**: Implement retry logic for network issues
3. **State sync**: Sync local state with conversation_id
4. **Error handling**: Graceful degradation on failures

### Testing Strategy

1. **Start simple**: Test with known good ADPCM files

### Custom Conversation Logic

Implement custom conversation handling:

```python
class ConversationManager:
    def __init__(self):
        self.history = []
        self.current_id = None
    
    def update(self, response):
        conv_id = response.get("conversation_id")
        conv_reset = response.get("conversation_reset")
        
        if conv_id != self.current_id:
            # New conversation
            self.history = []
            self.current_id = conv_id
        
        self.history.append({
            'timestamp': time.time(),
            'reset': conv_reset,
            'transcription': response.get('transcription'),
            'response': response.get('response')
        })
        
        return conv_reset

# Use with sensor
manager = ConversationManager()
resp = pt.upload(audio_data, format="adpcm")
is_reset = manager.update(resp)
```
